# 기업 소개 플랫폼 - 프로젝트 요구사항

> 작성일: 2026-05-22

---

## 프로젝트 개요

기업이 자신들의 정보를 소개하는 프로필을 등록하고,
방문자가 해당 정보를 열람할 수 있는 웹 플랫폼.
AI 챗봇을 통한 자연어 검색 및 기업 Q&A 기능 포함.

---

## 질의응답 기록

### Q1. 주요 사용자 설정
**질문:** 이 앱의 주요 사용자가 누구인가요?
**답변:** B — 기업이 직접 회원가입해서 프로필을 작성하고 관리

---

### Q2. 방문자 회원가입 필요 여부
**질문:** 방문자(열람자)는 회원가입이 필요한가요?
**답변:** A — 누구나 로그인 없이 볼 수 있음 (공개 열람)

---

### Q3. 기업 소개 글 형태
**질문:** 기업이 올리는 "소개 글"의 형태가 어떻게 되나요?
**답변:** A — 기업당 하나의 프로필 페이지 (단일 고정 페이지)

---

### Q4. 기업 프로필 포함 항목
**질문:** 기업 프로필 페이지에 들어가는 정보는 무엇인가요?
**답변:** 모든 항목 포함
- 회사명, 로고
- 회사 소개 (텍스트)
- 업종/카테고리
- 설립연도, 직원 수, 규모
- 웹사이트 링크
- 연락처 (이메일, 전화번호)
- 위치/주소
- 이미지/사진 갤러리
- SNS 링크 (인스타그램, 링크드인 등)

---

### Q5. 방문자 탐색 방식
**질문:** 방문자가 기업을 어떻게 탐색하나요?
**답변:** D — 검색 + 필터 + 카테고리 분류 모두 제공
- 회사명 검색
- 업종/지역별 필터링
- 카테고리(업종) 메뉴로 진입

---

### Q6. 운영자(관리자) 기능
**질문:** 운영자(관리자) 기능이 필요한가요?
**답변:** B — 기본 관리만
- 기업 가입 승인/거절
- 부적절한 프로필 삭제

---

### Q7. 프론트엔드 기술 스택
**질문:** 기술 스택은 어떻게 할까요?
**답변:** A — Nuxt.js + Tailwind CSS (Vue 기반)

---

### Q8. AI 챗봇 기능 범위
**질문:** AI 챗봇으로 무엇을 하고 싶으신가요?
**답변:** A + B
- A: 자연어 검색 ("IT 스타트업 서울에 있는 곳 찾아줘")
- B: 기업 Q&A ("OO기업은 어떤 회사야?" → 프로필 기반 답변)

---

### Q9. AI 구현 방식
**질문:** AI 챗봇 구현 방식을 어떻게 할까요?
**답변:** Ollama (로컬 LLM) + RAG 방식
- DB의 기업 프로필 데이터를 실시간으로 LLM에 주입
- 사용자 질문 → DB에서 관련 기업 검색 → 데이터를 LLM에 전달 → 자연어 답변 생성

---

### Q10. AI API 선택
**질문:** 무료 AI API 옵션 중 선택해주세요.
**답변:** Ollama (완전 무료, 로컬 설치형)

---

### Q11. 배포 환경
**질문:** 배포 환경이 어떻게 되나요?
**답변:** 로컬에서 시작 → 나중에 VPS 배포 결정
- 현재: 로컬 PC에서 Ollama + 웹앱 모두 실행
- 추후: 개인 서버/VPS에 배포 (별도 결정)

---

### Q12. Ollama 모델 선택
**질문:** Ollama에서 어떤 모델을 사용할까요?
**답변:** B — llama3.1:8b (5GB, 속도와 품질 균형, 한국어 지원)

---

### Q13. 백엔드 구성
**질문:** 백엔드를 어떻게 구성할까요?
**답변:** B — Nuxt.js(프론트) + FastAPI(백엔드) 분리 구조

---

### Q14. 데이터베이스
**질문:** 데이터베이스는 무엇을 사용할까요?
**답변:** B — PostgreSQL (로컬은 Docker로 실행)

---

### Q15. 인증 방식
**질문:** 기업 회원가입/로그인 인증 방식은 어떻게 할까요?
**답변:** B — 이메일 + 비밀번호 + 이메일 인증 (JWT 기반)

---

### Q16. 이미지 저장 방식
**질문:** 기업 로고 및 이미지는 어떻게 저장할까요?
**답변:** A — 서버 로컬 저장 (FastAPI StaticFiles 활용)
- 추후 배포 시 Cloudinary 또는 S3로 교체 예정

---

### Q17. UI 디자인 방향
**질문:** UI 디자인 방향은 어떻게 할까요?
**답변:** A — 깔끔한 비즈니스 스타일
- 흰 배경, 카드형 레이아웃, 전문적인 느낌
- 신뢰감과 전문성 강조

---

### Q18. 관리자 승인 흐름
**질문:** 관리자 승인 흐름은 어떻게 할까요?
**답변:** A — 승인 전까지 프로필 비공개
- 기업 가입 → 프로필 작성 → 관리자 검토 → 승인 후 공개

---

### Q19. 프로젝트 이름
**질문:** 이 플랫폼의 이름이 있나요?
**답변:** A — 아직 없음 (추후 결정)

## 최종 기술 스택 요약

| 구분 | 기술 |
|------|------|
| 프론트엔드 | Nuxt.js + Tailwind CSS |
| 백엔드 | FastAPI (Python) |
| 데이터베이스 | PostgreSQL (Docker) |
| 인증 | JWT + 이메일 인증 |
| AI | Ollama (llama3.1:8b) + RAG |
| 이미지 저장 | 서버 로컬 저장 (FastAPI StaticFiles) |

## 핵심 기능 요약

### 방문자
- 로그인 없이 기업 프로필 열람
- 키워드 검색 / 업종·지역 필터링 / 카테고리 탐색
- AI 챗봇으로 자연어 검색 및 기업 Q&A

### 기업 (회원)
- 이메일 회원가입 + 이메일 인증
- 프로필 페이지 작성 및 수정
- 승인 대기 → 관리자 승인 후 공개

### 관리자
- 기업 가입 승인/거절
- 부적절한 프로필 삭제

---

## 추후 결정 사항

- 프로젝트 이름
- VPS 배포 시기 및 환경
- 배포 후 이미지 스토리지 (Cloudinary or S3)
- Ollama → 클라우드 AI API 전환 여부
